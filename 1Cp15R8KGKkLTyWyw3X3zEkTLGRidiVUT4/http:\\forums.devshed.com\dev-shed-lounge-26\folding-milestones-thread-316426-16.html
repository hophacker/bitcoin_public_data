    Originally Posted by LinuxPenguin
    Then mount your windows partition as auto in your fstab, copy the linux FAH binary into the folder the windows one is in and set FAH to run from that directory. It will use the same WU as in windows then
    While the WU itself will run with either Win or Linux client, one cannot reliably bounce form one OS to the other and bakc on teh same WU that way-- windows sees the WU as corrupted when the Linux client has written to it and you will have to delete the WU.

    If you are going to Dual-boot and want to process the same WU in either OS, then you will want to a) place the folder holding the clinet and WU into a FAT32 partition for easy access by both OSes, and b) run the Win client under WINE in Linux.
